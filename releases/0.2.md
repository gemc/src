Description of changes and fixes in this release.

# Release notes

- Registry container names change and better CI summary
- Gemc version from the tag algorithm more robust
- Using assimp new release 6.0.5
- Examples added:
    - Flux
    - Materials
    - PyVista
    - Cherenkov

# New Developments

- Jupyter support, also added to the homepage
- LUND support for generator
- Analyzer Scripts
- `gemc.sh` environment script generated upon installation

# Issues Addressed

- [[#80]](https://github.com/gemc/src/issues/80): Add batch screenshot
- [[#81]](https://github.com/gemc/src/issues/81): Fix Cherenkov Bug (thanks to Zhiwen)


# Supported platforms

Both AMD and ARM processors are supported.

- macOS: 26
- Ubuntu: 24.04
- AlmaLinux: 9.4
- Fedora: 42
- Debian: 13
- ArchLinux: latest



# Dependencies

- Geant4: 11.4.1 or higher
- CLHEP: 2.4.6.0 or higher
- Xerces-C: 3.2 or higher



# Detailed list of changes and fixes

- docker images name change: removed Geant4 version
- removed Docker Build from the workflow summary
- using ci/gemc_version.sh for both the meson project version and the gemc version. 
  Uses the tag name to determine the version
- added optical properties and moved materials routines in g4materials.cc
- fixed CI tests not exiting if any failures
- fixed python api not resetting ascii files for new variations
- fixed g4view option help 
- added screenshot bool and function in GRunAction::EndOfRunAction
- added cloud style for volumes (0=wireframe, 1=solid, 2=cloud)
- various code cleanups
